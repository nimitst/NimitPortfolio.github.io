Nashville Housing Analysis

View the SQL Query for Nashville Housing Project

Working with a large Nashville housing dataset, I set out to transform unstructured, inconsistent data into a clean and reliable foundation for analysis. Using SQL Server, I began by standardising the date formats with the CONVERT() function and creating new columns to store the cleaned values. To address missing property addresses, I implemented a self-join on the ParcelID field, allowing records with null values to inherit the correct address data. I then used string functions such as SUBSTRING() and PARSENAME() to decompose full address fields into separate columns for street, city, and state. This improved data granularity and usability.

To enhance data consistency, I normalized categorical values in the SoldAsVacant field using CASE statements, converting ambiguous “Y” and “N” entries into clear “Yes” and “No” labels. I also applied a Common Table Expression (CTE) with the ROW_NUMBER() function to identify and remove duplicate records, ensuring data integrity. Finally, I streamlined the dataset by dropping redundant columns, leaving behind a clean, well-structured database ready for advanced analytics and reporting.
